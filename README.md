# Multimodal Patient-Specific Identification of Atrial Flutter Circuits From ECG Time Series Using Explainable Machine Learning

MATLAB pipeline for the classification of atrial flutter subtypes from the
standard 12-lead ECG, by means of:

1. ECG preprocessing and selection of the atrial (F-wave) segment,
2. Construction of the Vectorcardiogram (VCG) using the inverse Dower transform,
3. Extraction of geometric, velocity-based and template-correlation features
   from the averaged VCG loop,
4. Cascaded supervised classification of the four flutter subtypes:
   - **C_CW** &mdash; common (cavotricuspid isthmus dependent), clockwise
   - **C_CCW** &mdash; common, counter-clockwise
   - **PM_CW** &mdash; perimitral, clockwise
   - **PM_CCW** &mdash; perimitral, counter-clockwise

The classifier first separates *common* vs *perimitral* substrate; the
direction of rotation (*CW* vs *CCW*) is then resolved **within the substrate
that was correctly identified at the first stage**.

> [!IMPORTANT]
> **No patient data of any kind is included in this repository.**

The ECG recordings, VCG files, per-patient feature files, and clinical
spreadsheets used in the associated paper are **not distributed** here because

review and Spanish data-protection law. Access requests can be directed to

(ECG signals, VCG files, feature `.mat` files, any `.xlsx` spreadsheet with
clinical or signal data) is excluded by `.gitignore` and was never committed
to version control.

## Input data format

To run the pipeline on your own data you must provide the following files,
organised under `data/` (which is git-ignored):

### ECG files (`data/ECGs/`)

- One `.txt` file per patient.
- Each file contains the raw 12-lead ECG as a numeric matrix with one lead
  per **column** and one time sample per **row** (the loader calls
  `transpose` after reading, so the internal representation is 12 x N).
- Sampling frequency: **1000 Hz**.
- No header row.
- Naming convention:

  ```
  ECG_<id>_<group>_<direction>.txt
  ```

  where:

  | Token | Values | Meaning |
  |---|---|---|
  | `<id>` | positive integer | Unique numeric patient code |
  | `<group>` | `C` or `PM` | Substrate (common / perimitral) |
  | `<direction>` | `CW` or `CCW` | Rotation direction |

  Example: `ECG_80_C_CCW.txt`

### VCG files (`data/VCGs/`) — generated by script 01

Produced automatically by `scripts/01_preprocess_and_create_vcg.m`.
Each file is a 3-column `.txt` (X, Y, Z per row) with the averaged VCG
loop of one patient. Naming convention:

```
VCG_<id>_<group>_<direction>.txt
```

### Clinical data (optional, `data/features_master.xlsx`)

If you want to run the classifier with `USE_CLINICAL_DATA = true`, the
feature spreadsheet produced by script 04 must be manually enriched with
the clinical columns listed in `docs/FEATURES.md` (age, BMI,
CHA2DS2-VASc score, LVEF, etc.). These columns must be added externally
(e.g. by merging with the hospital dataset in Excel) before running
script 05. The required column names are documented in `docs/FEATURES.md`.

## Quick start

1. Clone the repository and open MATLAB in the project root.
2. Make sure your MATLAB installation includes the toolboxes listed in
   `REQUIREMENTS.md`.
3. Place your ECG files (see **Input data format** above) under `data/ECGs/`.

The repository supports two use cases:

### A) Train the classifier from scratch

Run the five orchestration scripts in order. The configurable parameters
(paths, patient code, flags) are at the top of each file.

```matlab
run('scripts/01_preprocess_and_create_vcg.m')   % once per patient
run('scripts/02_build_archetypes.m')
run('scripts/03_extract_patient_features.m')    % set SINGLE_PATIENT_CODE = []
run('scripts/04_consolidate_features.m')
run('scripts/05_train_classifier.m')
```

### B) Apply the trained pipeline to a new patient

```matlab
% Step 1 — build the new patient's VCG (set 'codigo' at the top of the script)
run('scripts/01_preprocess_and_create_vcg.m')

% Step 2 — extract all features (set SINGLE_PATIENT_CODE = <id>)
run('scripts/03_extract_patient_features.m')
```

The resulting `.mat` file can then be appended to the master spreadsheet
or fed directly into a trained classifier.

A detailed walk-through of every step is in `docs/PIPELINE.md`. A field
reference for every feature is in `docs/FEATURES.md`.

## Use of clinical variables

The final classification step can be run in two modes, controlled by a
single boolean flag (`USE_CLINICAL_DATA`) at the top of
`scripts/05_train_classifier.m`:

- **`USE_CLINICAL_DATA = true`** &mdash; the classifier uses both
  VCG-derived features and patient clinical variables (age, BMI,
  CHA2DS2-VASc, LVEF, etc.). This is the configuration reported in the
  paper.
- **`USE_CLINICAL_DATA = false`** &mdash; only VCG-derived features are
  used. This mode requires no clinical data and can be applied to any new
  patient for whom only the ECG is available.

The clinical spreadsheet is **not** distributed with this repository; it
is held under restricted access at the hospital of origin. See
`docs/FEATURES.md` for the required column names and coding conventions.
